# 渐进式阅读体验增强设计

日期：2026-03-28

## 背景

RssHub 是一个个人 AI 驱动的 RSS 阅读器，聚焦垂直深读场景。当前已有评分、翻译、摘要等 AI 服务代码，但阅读 pipeline 未打通。用户需求：先优化阅读体验，再做知识管理。

## 设计方案

### 阶段 1：打通阅读触发器

**问题**：`items.py` 中 `update.status == "reading"` 的触发器是空实现（`pass`）。

**方案**：标记 reading 时异步执行 pipeline：

```
标记 reading → 1. 抓取全文 (content_fetcher)
             → 2. 翻译全文 (translator, 如果是英文)
             → 3. 生成 AI 摘要 (summarizer)
             → 4. 更新 Item 记录 (content, content_zh, summary_ai)
             → 5. 记录偏好 (preference)
```

- 静默处理：用 `asyncio.create_task()` 异步执行，不阻塞 API 响应
- 每步独立 try/catch，单步失败不阻塞后续（如翻译失败仍保留原文）
- `discarded` 时也记录偏好学习（反向信号）

### 阶段 2：阅读列表页 + 四状态流

**状态流**：
```
inbox → reading → read
                 → discarded
```

- `inbox`：待筛选
- `reading`：已标记要读
- `read`：已读完（为知识管理打基础）
- `discarded`：不感兴趣

**新增路由**：
- `GET /reading`：阅读列表页，展示 `reading` 状态文章
- `GET /history`：已读历史（可选，后续实现）

**阅读列表页功能**：
- 按分数降序展示 `reading` 状态文章
- 显示：中文标题、分数、发布日期、AI 摘要
- 操作：进入详情、标记已读、丢弃

**文章详情页变更**：
- 增加"标记已读"按钮

### 阶段 3：AI 阅读辅助

**数据模型变更**：
- `key_points` (Text)：JSON 格式，存储 AI 提取的 3-5 个关键要点
- `read_time_minutes` (Integer)：预估阅读时长

**功能**：
- AI 从全文提取关键要点（bullet list）
- 阅读时长估算：中文 ~400 字/分钟，英文 ~250 词/分钟
- 详情页在摘要和全文之间展示 AI 信息区

**触发时机**：阶段 1 的 pipeline 中，全文抓取完成后一并生成。

### 阶段 4：移动端响应式优化

**方案**：在现有 `base.html` CSS 中增加 media query，保持轻量。

**优化内容**：
- 导航栏：移动端折叠为汉堡菜单
- Inbox / 阅读列表：卡片式布局，触摸友好按钮
- 文章详情：阅读模式优化（字号、行距、边距）

## 技术决策

1. **静默处理而非进度反馈**：个人使用场景，简单优先
2. **四状态流**：区分"要读"和"已读"，为知识管理铺路
3. **简化 AI 辅助**：关键要点 + 阅读时长，暂不做关联推荐
4. **纯 CSS 响应式**：不加前端框架，保持轻量
